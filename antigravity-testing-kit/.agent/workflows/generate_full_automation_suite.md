Explore the application and generate a complete automation testing suite.

Instructions:

1. Open the provided application URL.
2. Explore the application structure and navigation.
3. Identify major modules and user flows.
4. Generate test scenarios for critical flows.
5. Generate manual test cases.
6. Generate automation test skeletons.

Common flows to discover:

- Login and authentication
- User registration
- Search functionality
- Product browsing
- Add to cart
- Checkout process
- Profile management
- Logout

Automation framework:

Language: Java  
UI Automation: Selenium WebDriver  
Test Framework: TestNG  
Design Pattern: Page Object Model  

Locator strategy:

1. id  
2. data-testid  
3. name  
4. css selector  
5. xpath  

Output:

1. Application overview
2. Identified modules
3. Test scenarios
4. Manual test cases
5. Automation test skeleton

Automation output should include:

- Page Object classes
- Test classes
- Assertions validating expected behavior
