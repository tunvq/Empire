Instructions:

1. Analyze the provided feature or test case.
2. Identify input fields and validation rules.
3. Generate realistic test data sets.

Include the following categories:

Positive data  
Negative data  
Boundary values  
Edge cases  

Examples:

- valid email
- invalid email format
- empty input
- maximum length input
- special characters

Output:

Structured test data sets organized by category.

Format:

Positive test data  
Negative test data  
Boundary values  
Edge cases
